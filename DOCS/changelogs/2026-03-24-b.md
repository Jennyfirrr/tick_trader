# Changelog: March 24, 2026 (Critical: Heap Corruption Fix)

## [3.0.19] - 2026-03-24

### Critical Fix
- **Heap corruption crash from OpenSSL alloc/free churn** — The engine crashed with "corrupted size vs. prev_size" / "corrupted unsorted chunks" during REST API calls. Root cause: `SSL_new`/`SSL_free` on every request fragmented the heap until glibc detected corrupted metadata.

### Fixed
- **Zero-alloc SSL path** — `SSL_CTX` created once at startup. `SSL` object created once and reused via `SSL_clear()` (resets session state without heap alloc/free). TCP socket is the only thing cycled per-request (kernel-managed, no userspace heap impact).
- **Same-tick sell+buy guard** — Live buy orders are deferred when a sell happened in the same slow-path tick. Paper position is undone by existing orphan logic, BuyGate fires naturally next cycle.
- **Balance sync guard** — Failed REST balance query (HTTP -1) no longer zeros out paper balance or triggers false external-sell/orphan detection. Sync is skipped and retried next slow-path cycle.
- **Socket read timeout** — 5-second `SO_RCVTIMEO` on REST sockets prevents `SSL_read` from blocking indefinitely.

### Added
- **External sell detection** — Engine now detects when BTC was sold outside the engine (e.g. Binance app/website). If paper positions exist but exchange BTC balance is near zero, paper positions are undone and balance synced to actual USDT. Detected within one slow-path cycle (~30s) instead of requiring a restart.

### Rolled Back (then superseded)
- **Keep-alive REST connections** — Attempted but reverted. HTTP response parser doesn't handle keep-alive properly. Superseded by SSL object reuse via `SSL_clear` which achieves the same goal (zero heap churn) without requiring keep-alive.

### Changed
- **README** — Updated test counts (101 assertions, 27 test functions). Added manual trade reconciliation note. Added personal use clarification. Added options data donation goal.
