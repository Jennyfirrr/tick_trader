# Changelog: March 24, 2026 (Critical: Heap Corruption Fix)

## [3.0.19] - 2026-03-24

### Critical Fix
- **Heap corruption crash on back-to-back REST calls** — The engine crashed with "corrupted size vs. prev_size" when a SELL and BUY fired in the same slow-path tick. Root cause: full SSL/TLS teardown and recreation on every REST request caused OpenSSL alloc/free churn that corrupted glibc heap metadata.

### Fixed
- **Persistent SSL_CTX** — TLS context is now created once at startup and reused across all REST connections. Eliminates the heaviest source of heap allocation churn (cipher lists, cert chains, session cache). SSL objects are still created/destroyed per-request (lightweight).
- **Same-tick sell+buy guard** — Live buy orders are deferred when a sell happened in the same slow-path tick. Paper position is undone by existing orphan logic, BuyGate fires naturally next cycle. Defense-in-depth against back-to-back REST calls.
- **SSL_CTX_free removed from error paths** — Only `BinanceOrderAPI_Cleanup` frees the TLS context. Prevents NULL dereference if an error triggers mid-session reconnect.
- **Balance sync guard** — Failed REST balance query (HTTP -1) no longer zeros out paper balance or triggers false external-sell/orphan detection. Sync is skipped and retried next slow-path cycle.

### Added
- **External sell detection** — Engine now detects when BTC was sold outside the engine (e.g. Binance app/website). If paper positions exist but exchange BTC balance is near zero, paper positions are undone and balance synced to actual USDT. Detected within one slow-path cycle (~30s) instead of requiring a restart.

### Rolled Back
- **Keep-alive REST connections** — Reverted to close-after-request. The HTTP response parser doesn't handle keep-alive properly (no chunked transfer support, potential SSL_read blocking). All slow-path REST calls were failing with HTTP -1 after the first successful startup query. Close-after-request is reliable; the persistent SSL_CTX still eliminates the heavy alloc churn.
- **Connection staleness detection** — Removed (not needed with close-after-request).

### Changed
- **README** — Updated test counts (101 assertions, 27 test functions). Added note about manual trades on Binance app requiring engine restart to reconcile.
