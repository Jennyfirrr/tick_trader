# Changelog: March 24, 2026 (TP/SL Audit + TUI Improvements)

## [3.0.21] - 2026-03-24

### Fixed (Critical)
- **TRENDING_DOWN TP/SL asymmetry** — Regime adjustment used `take_profit_pct*100` (MR's 4σ) instead of `momentum_tp_mult` (3σ) for momentum positions entering downtrend. This created a 4:1 TP/SL ratio against the trader — SL at 1σ triggered on routine noise while TP at 4σ was unreachable. Root cause of 20% win rate / 0.05 profit factor in bear markets. Now uses correct config field with 2:1 minimum reward/risk floor.
- **SL floor missing on regime adjustment** — All three regime transition paths (RANGING→TRENDING, TRENDING→RANGING, →TRENDING_DOWN) could violate the 2:1 minimum reward/risk ratio established at fill time. SL floor re-check (SL distance >= 0.5 × TP distance) now enforced after every TP/SL mutation in `Regime_AdjustPositions`.
- **stddev=0 regime adjustment crash** — If regime switched during a flat market (stddev=0), all offsets became zero, setting TP=SL=entry and triggering immediate exit. Early return guard added, matching existing pattern in `Momentum_ExitAdjust` and `MeanReversion_ExitAdjust`.
- **Unguarded FPN_DivNoAssert** — 4 division sites lacked zero-denominator guards: `fill_price` in position sizing, `entry_price` in time-exit, `price_avg` in MR and Momentum buy signals. FPN_DivNoAssert saturates to MAX on zero, silently producing extreme values.

### Added
- **Half-block area chart** — Price chart uses Unicode half-block characters (▄█) for 2× vertical resolution: 3 terminal rows = 6 vertical levels with solid filled area below the data line. Green when rising, red when falling. Much more readable than sparkline (1 row / 8 levels) while showing trend direction at a glance.
- **Buy gate direction indicator** — TUI buy gate section shows `>=` for momentum (buy above / breakouts) and `<=` for mean reversion (buy below / dips). Previously hardcoded to `<=` regardless of active strategy. Updated across all 4 TUI backends (ANSI, FTXUI, notcurses, legacy printf).
- **Headless auto-logging** — Engine automatically redirects stderr to `log_file` (default: `engine.log`). Rotates on startup: `engine.log` → `engine.log.1`. Line-buffered for `tail -f`. Works in both TUI and headless mode — no more manual `2>engine.log`.
- **Wall-time slow-path floor** — Slow path now runs after 100 ticks OR 3 seconds, whichever comes first. Previously tick-only gating meant low-volume periods (1-2 trades/sec) could stretch slow-path cycles to 60+ seconds, stalling warmup, regime detection, and graph sampling.
- **Safety invariants section in CLAUDE.md** — Codifies rules for TP/SL mutations (preserve floors), FPN division guards (always check zero), config field conventions (_pct vs _mult), and regime adjustment checklist. Prevents the class of bugs found in this audit.
- **Strategy addition guide in CLAUDE.md** — Step-by-step checklist for adding new strategies: define constant, create header with 4 functions (_Init, _Adapt, _BuySignal, _ExitAdjust), wire into PortfolioController dispatch, wire into RegimeDetector mapping.
- **17 new test assertions** (166 total) — Regression tests for TRENDING_DOWN TP/SL ratio, stddev=0 guard, and SL floor invariant on all 4 regime transition paths.

### Changed
- **`tui_enabled` in BinanceConfig** — Parsed from engine.cfg, used by TUI_Init instead of hardcoded `1`. Resolves existing TODO.
- **`log_file` config option** — New field in BinanceConfig (default: `engine.log`). Set empty to disable auto-logging.
- **Graph minimum threshold** — Charts appear after 2 data points instead of 4 (~6 seconds sooner).

### Technical Notes
- **Zero hot-path impact** — All changes are on slow path, fill-consumption path (runs ~5×/session), or TUI thread. BuyGate and PositionExitGate untouched. Verified via `LATENCY_PROFILING` — p50 unchanged.
- **Rollback point** — `git tag pre-audit-fixes` created before changes. Each logical phase is independently revertible.
- **12 files changed**, 353 insertions, 19 deletions across RegimeDetector, PortfolioController, 4 TUI backends, main.cpp, BinanceCrypto, ControllerConfig, engine.cfg.example, CLAUDE.md, tests.
