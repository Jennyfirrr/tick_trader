## [3.0.12] - 2026-03-22

### Fixed
- **Snapshot load skipped warmup** — when positions existed in snapshot, engine set
  CONTROLLER_ACTIVE immediately, trading with empty rolling stats. Now always runs
  warmup regardless of loaded positions. Loaded positions keep their TP/SL exits
  (exit gate runs every tick).
- **Exit buffer not drained during warmup** — extracted drain loop into
  `PortfolioController_DrainExits()` helper, called from both warmup and slow path.
  Loaded positions that hit TP/SL during warmup get processed immediately.

### Changed
- **TUI reverted to 9-section layout** — user preferred expanded view with separate
  PORTFOLIO, P&L, RISK, CONFIG, STATS sections. Kept all improvements: W/L in TopBar,
  fox art (ド ヘ), BUYING PAUSED indicator, fill rejection diagnostics, dynamic warmup
  sample display.

## [3.0.10] - 2026-03-22

### Fixed
- **Warmup too short** — `min_warmup_samples=20` let trading start before the short and
  long windows had divergent data (both had identical R²). Bumped to 128 so the short
  window fills completely before trading (~71 min at BTC rate). Regime detector now has
  genuinely different timeframes to compare.
- **Version strings stale** — TUI showed "engine v0.6/v0.7/v0.8" across backends.
  Updated all three renderers (ANSI, notcurses, legacy printf) to v3.0.9.
- **Hardcoded warmup display** — TRADING BLOCKED showed "(%d/20 samples)" regardless
  of config. Now reads `min_warmup_samples` from snapshot so it reflects whatever the
  user configures.

## [3.0.7] - 2026-03-22

### Fixed
- **Buy spam on restart** — warmup pushed to rolling stats every tick, flooding the
  128-sample window with 43 seconds of near-identical prices (stddev≈$0.33 on $68k).
  R²=1.000 on 2 slow-path samples → false TRENDING classification → 10 positions
  opened in 19 seconds.
  - **Root cause**: two concerns tangled in one config value. `warmup_ticks` counted raw
    ticks (fast), but meaningful stats need slow-path-spaced samples (real price diversity).
  - **Fix**: warmup now pushes to rolling stats at slow-path rate
    (`warmup_count % poll_interval == 0`). Each sample represents a real time interval,
    not a raw tick.
  - **New config**: `min_warmup_samples` (default 0 in code, 20 in engine.cfg) gates
    warmup exit on having enough slow-path samples for meaningful regression.
  - Tests use `poll_interval=1` → every tick IS a slow-path push → behavior unchanged.
  - Production uses `poll_interval=100` → 20 samples × 100 ticks ≈ 11 min of market data
    before first buy.

### Added
- `test_warmup_ctrl()` helper function for future tests — auto-computes tick count from
  config values (`warmup_ticks`, `min_warmup_samples`, `poll_interval`).
