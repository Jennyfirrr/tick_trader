## [3.0.7] - 2026-03-22

### Fixed
- **Buy spam on restart** — warmup pushed to rolling stats every tick, flooding the
  128-sample window with 43 seconds of near-identical prices (stddev≈$0.33 on $68k).
  R²=1.000 on 2 slow-path samples → false TRENDING classification → 10 positions
  opened in 19 seconds.
  - **Root cause**: two concerns tangled in one config value. `warmup_ticks` counted raw
    ticks (fast), but meaningful stats need slow-path-spaced samples (real price diversity).
  - **Fix**: warmup now pushes to rolling stats at slow-path rate
    (`warmup_count % poll_interval == 0`). Each sample represents a real time interval,
    not a raw tick.
  - **New config**: `min_warmup_samples` (default 0 in code, 20 in engine.cfg) gates
    warmup exit on having enough slow-path samples for meaningful regression.
  - Tests use `poll_interval=1` → every tick IS a slow-path push → behavior unchanged.
  - Production uses `poll_interval=100` → 20 samples × 100 ticks ≈ 11 min of market data
    before first buy.

### Added
- `test_warmup_ctrl()` helper function for future tests — auto-computes tick count from
  config values (`warmup_ticks`, `min_warmup_samples`, `poll_interval`).
