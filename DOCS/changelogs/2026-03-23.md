# Changelog: March 23, 2026 (Latency & Profiling Overhaul)

## [1.0.2] - 2026-03-23

### Fixed
- **Warmup Visibility Bug** — Latency profiling now shows "hot path" ticks immediately during warmup (was incorrectly tagging all ticks as slow path due to `tick_count` skip).
- **Inflated PC Metrics** — Moved `MetricsLog_Event` (file I/O) outside the RDTSCP profiling window, dropping recorded `pc` latency from ~2000ns to <1000ns (and <100ns in ACTIVE phase).
- **ExitGate Noise** — Tightened measurement window to exclude variable setup, fixing the 1000ns "ghost" latency seen when 0 positions were active.

### Added
- **Native 128-bit Math** — New `profile-fast` target enables `USE_NATIVE_128`, switching from generic software loops to hardware-native `__uint128_t` for all price/volume checks.
- **Busy Polling Support** — `BUSY_POLL=ON` flag for `profile-fast` enables spin-wait with `pause` hint, eliminating OS sleep-wake jitter (shaves ~500-1000ns off `p95` latency).
- **Core Isolation** — Engine pinned to Core 3, TUI to Core 2 (via `pthread_setaffinity_np`) to isolate execution from system interrupts on Core 0.

### Changed
- **Hot-Path De-bloating** — Moved session high/low tracking and fixed-to-double conversions out of the active hot-path to ensure the engine logic is as lean as possible.
- **Makefile Refactor** — Added `profile-fast` target with optional `BUSY_POLL=ON` toggle.
