# Changelog: March 24, 2026 (Single-Slot Position Management)

## [3.0.18] - 2026-03-24

### Added
- **`max_positions` config** — Caps simultaneous open positions (1-16, default 1). Hot-reloadable. Replaces the old `Portfolio_IsFull` (16-slot) check with a configurable limit.
- **Paper position undo** — When a live buy is skipped (below min_notional) or fails (API error), the paper position is immediately reversed: bitmap bit cleared, balance restored. Prevents paper/live balance drift.
- **Sell entire BTC balance** — In single-slot mode (`max_positions=1`), exit sells the full Binance BTC balance instead of the paper-tracked quantity. Eliminates dust accumulation from quantity rounding.
- **Startup BTC recovery** — On startup, if orphaned BTC exists on the exchange with no matching snapshot positions, it's sold automatically before trading begins. No manual dust management needed.
- **24h reconnect sweep** — Session boundary sell now uses full BTC balance in one order instead of walking per-position quantities. Catches tracked positions + any accumulated dust.
- **Multi-slot sell guard** — Sell-all-BTC only fires when `max_positions==1`. Falls back to per-position sells for `max_positions > 1` to avoid accidentally dumping other positions' BTC.
- **6 new test assertions** (140 total) — `max_positions=1` rejects second fill, `max_positions=2` accepts two and rejects third, default config check.

### Fixed
- **`slippage_pct` missing from hot-reload** — Changing slippage in engine.cfg and pressing 'r' now takes effect immediately (was silently ignored until restart).
- **TUI reject label** — Fill rejection reason "full" renamed to "max_pos" to reflect the new position cap behavior.

### Changed
- **README dust note** — Updated to reflect that single-slot mode eliminates dust automatically. Notes that `max_positions > 1` may still accumulate dust.
- **DOCS/CONFIGURATION.md** — Added `max_positions` to the config reference table and example.
