# Changelog: March 24, 2026 (Direction-Aware Regime Detection)

## [3.0.20] - 2026-03-24

### Added
- **REGIME_TRENDING_DOWN** — New regime constant for downtrends. The regime detector now distinguishes between uptrends (REGIME_TRENDING → momentum) and downtrends (REGIME_TRENDING_DOWN → pause buying). Previously, `FPN_Abs()` on slopes made downtrends score identically to uptrends, dispatching momentum which bought dead cat bounces.
- **Direction-aware classifier** — `Regime_Classify` tracks `up_signals` and `down_signals` separately. Slopes, ROR, and volume each contribute to the appropriate direction. Requires at least one slope signal to classify as trending (prevents R² + ROR alone from triggering trending).
- **TUI labels** — All three TUI variants (ANSI, FTXUI, notcurses) display "TR_DN" / "TRENDING_DOWN" in red for the new regime. Buying paused indicator shows "downtrend — buying paused".
- **Manual regime cycle** — `s` key now cycles: RANGING → TRENDING → TRENDING_DOWN → VOLATILE → RANGING.

### Changed
- **Regime_ToStrategy** — TRENDING_DOWN maps to STRATEGY_MEAN_REVERSION (extensibility point for future short strategy).
- **Position adjustment** — Entering TRENDING_DOWN tightens TP (take profits) and tightens SL (cut losses). Exiting trending down to RANGING uses same adjustment as exiting TRENDING.
- **Pause block** — TRENDING_DOWN pauses buying same as VOLATILE. No entries in downtrends until a short strategy is implemented.
- **TradeLog + metrics** — `_regime_str` returns "TRENDING_DOWN" for regime 3.

### Limitations
- TRENDING_DOWN currently just pauses — no active downtrend strategy. This is the extensibility point for a future short strategy.
- Snapshot compatibility preserved — old snapshots without regime 3 default to RANGING on load (existing behavior).
