# Changelog: March 23, 2026 (Latency Reduction + Slippage Simulation)

## [1.0.3] - 2026-03-23

### Added
- **Slippage Simulation** — new `slippage_pct` config parameter simulates worse execution on both entry (buy higher) and exit (sell lower). Applied in all three exit paths: TP/SL drain, time-based exit. Default 0 (disabled, backward compatible). Set `slippage_pct=0.05` in engine.cfg for 0.05% round-trip slippage.
- **Live Trading Plan** — `plans/live_trading_plan.md` documents the design for Binance REST API integration, order state machine, safety interlocks, and testnet-first strategy.
- **Slippage Tests** — 4 new assertions in controller_test: buy slippage, sell slippage P&L, and disabled (backward compat). Total: 134 assertions.

### Changed
- **Zero-Pollution TUI Snapshot** — full `TUI_CopySnapshot` moved from every-32-ticks to the slow path, where L1 is already warm from rolling stats, regime, strategy dispatch. Zero additional cache eviction on the hot path. Live price/volume/active_count update every tick via stashed doubles (1 cache line, ~7ns).
- **Stashed Doubles in DataStream** — `atof(price_str)` runs alongside `FPN_FromString` during websocket parse (hidden in I/O-bound code). Stored as `price_d`/`volume_d` in DataStream struct. Eliminates all `FPN_ToDouble(last_stream.price)` calls in the engine loop.
- **Warmup Hot-Path Trim** — moved `FPN_ToDouble`, `FPN_AddSat` (session tracking, price/volume accumulation) from every-tick to every-`poll_interval`-tick during warmup. Saves ~50-60ns per warmup tick. `warmup_count++` remains every-tick for completion gate.
- **Branch Prediction Hint** — added `__builtin_expect(ctrl->state == CONTROLLER_WARMUP, 0)` to hint the CPU that warmup is the unlikely path after warmup completes.

### Performance Impact
- Hot-path avg with TUI on should approach bench-mode levels (~100-200ns vs previous ~1000-1600ns)
- Price display updates every tick (more responsive than before)
- Balance, regime, positions update every slow-path cycle
- Graph sparklines append on slow-path cycles (sparser, but data only changes then anyway)
- No change to bench mode (TUI off) numbers — already optimal

### Config
New parameters in engine.cfg:
```
slippage_pct=0.05    # simulated slippage (%, 0.05 = 0.05% worse on entry/exit, 0 = disabled)
use_real_money=0     # 0=paper trading (default), 1=real orders via Binance REST API
```

## [1.0.4] - 2026-03-23

### Added
- **Binance REST Order API** — `DataStream/BinanceOrderAPI.hpp` implements authenticated
  order placement via Binance REST API. HMAC-SHA256 signing, market buy/sell, fill price
  parsing, exchange filter validation (LOT_SIZE, minNotional), retry with exponential
  backoff, balance query, clock sync. Reuses TCP/TLS patterns from BinanceCrypto.hpp.
- **Fire-and-Forget Live Trading** — set `use_real_money=1` in engine.cfg to place real
  orders. Paper logic runs first (all validation gates), then matching REST order fires.
  No pending order tracking — market orders fill in <200ms. `live_position_bitmap`
  (uint16_t) tracks which paper slots have real Binance backing.
- **Balance Sync** — Binance USDT balance queried every slow-path cycle and synced to
  paper balance. Binance is source of truth. Auto-corrects for fees and drift.
- **Orphan Detection** — `live_position_bitmap & ~active_bitmap` detects real positions
  without paper backing. Orphans are auto-sold and logged.
- **24h Reconnect Sell-All** — before the 24h session reconnect, all live positions are
  sold on Binance. Bitmap cleared. Paper force-close proceeds as normal.
- **Startup Reconciliation** — on init, balance is synced from Binance and all snapshot
  positions are marked as live (conservative).
- **Quantity Validation** — LOT_SIZE stepSize and minNotional filters queried from
  `/api/v3/exchangeInfo` at init. Quantities truncated to stepSize (branchless int cast).
  Orders below minNotional are skipped with a log.
- **Binance US Support** — `use_binance_us=1` routes REST orders to `api.binance.us`.
  Testnet blocked from US, but Binance US API works.
- **Lightweight Status Line** — one stderr line per slow-path cycle in live mode:
  `[LIVE] $70495.85 RANGE pos:2/16 live:2 bal:$9854.63 W:15 L:36`

### Architecture
- All REST interaction in main.cpp — PortfolioController stays pure paper logic, zero
  coupling, tests unaffected (134/134 passing).
- Paper fill runs first (validates spacing, balance, exposure, circuit breaker), then
  real order submitted to match. No rollback on failure — paper position kept, bitmap
  bit not set. Position is paper-only if REST fails.
- Sell buffer saved before PCTick (config-constant branch, ~0ns in paper mode). Sells
  fire after PCTick, bitmap-gated (only for positions with real backing).
- Hot path (BuyGate, ExitGate, PCTick) completely untouched. Zero new branches inside.

### How to Get API Keys (US Users)
- **Binance US** (recommended for US): Create account at binance.us → Account Settings
  → API Management → Create API key (enable Spot Trading permission)
- **Binance Testnet**: https://testnet.binance.vision (blocked from US IPs — need VPN)
- **Binance Global**: Create account at binance.com (not available in US)

### Setup
1. Get API keys from Binance US (or testnet with VPN)
2. Create `secrets.cfg` in project root:
   ```
   api_key=your_key_here
   api_secret=your_secret_here
   ```
3. Set in `engine.cfg`:
   ```
   use_binance_us=1
   use_real_money=1
   ```
4. Build and run: `cmake -B build && cmake --build build && cd build && ./engine 2>engine.log`
5. Monitor: `tail -f engine.log` + Binance dashboard for positions
