# Changelog: March 23, 2026 (Latency Reduction + Slippage Simulation)

## [1.0.3] - 2026-03-23

### Added
- **Slippage Simulation** — new `slippage_pct` config parameter simulates worse execution on both entry (buy higher) and exit (sell lower). Applied in all three exit paths: TP/SL drain, time-based exit. Default 0 (disabled, backward compatible). Set `slippage_pct=0.05` in engine.cfg for 0.05% round-trip slippage.
- **Live Trading Plan** — `plans/live_trading_plan.md` documents the design for Binance REST API integration, order state machine, safety interlocks, and testnet-first strategy.
- **Slippage Tests** — 4 new assertions in controller_test: buy slippage, sell slippage P&L, and disabled (backward compat). Total: 134 assertions.

### Changed
- **Zero-Pollution TUI Snapshot** — full `TUI_CopySnapshot` moved from every-32-ticks to the slow path, where L1 is already warm from rolling stats, regime, strategy dispatch. Zero additional cache eviction on the hot path. Live price/volume/active_count update every tick via stashed doubles (1 cache line, ~7ns).
- **Stashed Doubles in DataStream** — `atof(price_str)` runs alongside `FPN_FromString` during websocket parse (hidden in I/O-bound code). Stored as `price_d`/`volume_d` in DataStream struct. Eliminates all `FPN_ToDouble(last_stream.price)` calls in the engine loop.
- **Warmup Hot-Path Trim** — moved `FPN_ToDouble`, `FPN_AddSat` (session tracking, price/volume accumulation) from every-tick to every-`poll_interval`-tick during warmup. Saves ~50-60ns per warmup tick. `warmup_count++` remains every-tick for completion gate.
- **Branch Prediction Hint** — added `__builtin_expect(ctrl->state == CONTROLLER_WARMUP, 0)` to hint the CPU that warmup is the unlikely path after warmup completes.

### Performance Impact
- Hot-path avg with TUI on should approach bench-mode levels (~100-200ns vs previous ~1000-1600ns)
- Price display updates every tick (more responsive than before)
- Balance, regime, positions update every slow-path cycle
- Graph sparklines append on slow-path cycles (sparser, but data only changes then anyway)
- No change to bench mode (TUI off) numbers — already optimal

### Config
New parameter in engine.cfg:
```
slippage_pct=0.05    # simulated slippage (%, 0.05 = 0.05% worse on entry/exit, 0 = disabled)
```
